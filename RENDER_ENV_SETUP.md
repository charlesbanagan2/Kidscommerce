## Troubleshooting

### Still Getting 500 Error?

**Check Render Logs:**
1. Go to Render Dashboard
2. Click your service
3. Click **Logs** (top right)
4. Look for error messages

**Common Issues:**
- **SMTP Authentication Error:** Gmail App Password is invalid
  - Solution: Generate new App Password (see REGISTRATION_500_EMAIL_ERROR.md)
- **Connection Refused:** Network issue
  - Solution: Redeploy service
- **Missing Variable:** Typo in variable name
  - Solution: Double-check spelling (case-sensitive!)

### Email Not Sending?

**Test Gmail App Password:**
1. Try logging in to Gmail with the app password
2. If it fails, generate a new one:
   - Go to https://myaccount.google.com/
   - Security → 2-Step Verification → App passwords
   - Generate new password for "Mail"
   - Update `MAIL_APP_PASSWORD` in Render.com

### How to Update a Variable?

1. Go to Render Dashboard → Your Service → Environment
2. Find the variable you want to update
3. Click the **Edit** icon (pencil)
4. Update the value
5. Click **Save Changes**
6. Service will redeploy automatically

## Important Notes

⚠️ **Security:**
- Never commit `.env` file to GitHub
- Keep credentials secure
- Rotate passwords regularly

⚠️ **Gmail App Password:**
- This is NOT your regular Gmail password
- It's a 16-character password generated by Google
- Used for apps that don't support 2-Step Verification

⚠️ **Render.com:**
- Free tier sleeps after 15 minutes of inactivity
- First request after sleep takes 30-60 seconds
- Consider upgrading to paid tier for production
