EMAIL DUPLICATE REGISTRATION FIX - SUMMARY
================================================================================

PROBLEMA:
---------
Nagkaroon ng dalawang user na may same email kasi yung una ay "pending" pa 
ang status (hindi pa approved), tapos nag-register ulit ng same email.

TRANSLATION:
Two users with the same email were created because the first one had "pending" 
status (not yet approved by admin), then someone registered again with the 
same email, causing duplicate registrations.


SOLUTION:
---------
✅ Updated backend /api/check-email endpoint
   - Now specifically checks for "pending" status
   - Returns clear message: "waiting for admin approval"
   - Includes status field in response

✅ Updated backend /api/register endpoint  
   - Double protection during registration
   - Blocks pending emails with specific message
   - Allows re-registration for rejected users

✅ Updated frontend register_screen.dart
   - Checks status field from API
   - Shows user-friendly error message
   - Real-time validation as user types

CHECK FOR DUPLICATES:
---------------------
Run this SQL query:

  SELECT email, COUNT(*) as count, 
         STRING_AGG(status, ', ') as statuses
  FROM "user"
  GROUP BY email
  HAVING COUNT(*) > 1;


HOW IT WORKS NOW:
-----------------
1. User enters email → System checks in real-time
2. If email is pending → Shows error immediately
3. User cannot proceed → No duplicate created ✅


ERROR MESSAGES:
---------------
Pending:  "This email is waiting for admin approval. Please wait or contact support."
Active:   "This email is already registered. Please use a different email or login."
Rejected: Allows re-registration
New:      Proceeds normally
