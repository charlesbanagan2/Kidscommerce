DATABASE STRUCTURE
================================================================================

Table: chat_message
------------------------------------------------------------
Column          Type                Constraints
------------------------------------------------------------
id              INTEGER             PRIMARY KEY (auto-increment)
sender_id       INTEGER             NOT NULL, FK -> user.id
receiver_id     INTEGER             NOT NULL, FK -> user.id
message         TEXT                NOT NULL
is_read         BOOLEAN             DEFAULT false
created_at      TIMESTAMP           DEFAULT CURRENT_TIMESTAMP

Indexes:
------------------------------------------------------------
- idx_chat_sender          (sender_id)
- idx_chat_receiver        (receiver_id)
- idx_chat_created         (created_at DESC)
- idx_chat_is_read         (is_read) WHERE is_read = FALSE
- idx_chat_conversation    (sender_id, receiver_id, created_at DESC)

================================================================================
API ENDPOINTS
================================================================================

All endpoints require JWT authentication (Bearer token in Authorization header)

1. GET /api/chat/conversations
   - Returns all conversations for the current user
   - Includes unread count per conversation
   - Sorted by most recent message

2. GET /api/chat/messages/<user_id>
   - Returns all messages with a specific user
   - Automatically marks messages as read
   - Sorted chronologically

3. POST /api/chat/send
   - Send a message to another user
   - Body: { "receiver_id": <int>, "message": "<string>" }
   - Returns the created message

4. GET /api/chat/unread-count
   - Returns total unread message count for current user

================================================================================
REAL-TIME FEATURES (Socket.IO)

Events:
- join_chat: User joins their personal chat room
- new_message: Real-time message delivery

- user_stop_typing: Receive stop typing notification
