========================================

## ROOT CAUSE IDENTIFIED ✅

**Problem**: Supabase REST API returns EMPTY results for products
- PostgreSQL database: Product 17 EXISTS with 50 stock ✅
- Supabase REST API: Returns [] (empty) ❌

**Why**: Supabase REST API has Row Level Security (RLS) or permissions blocking access

**Impact**: get_available_stock() function returns 0 for ALL products

## SOLUTION APPLIED ✅

**Changed get_available_stock() to use SQLAlchemy ORM instead of Supabase REST API**

### Before (BROKEN):
```python
products = get_data('product', filters={'id': product_id})  # Returns []
if not products:
    return 0  # Always returns 0!
```

### After (FIXED):
```python
product = db.session.get(Product, product_id)  # Direct database access
if not product:
    return 0
# Returns actual stock!
```

## WHAT WAS CHANGED ✅

**File**: backend/app.py
**Function**: get_available_stock (line ~2356)

**Changes**:
1. Replaced `get_data('product', ...)` with `db.session.get(Product, ...)`
2. Replaced `get_data('order_item', ...)` with `OrderItem.query.filter_by(...).all()`
3. Replaced `get_data('restock_request', ...)` with `RestockRequest.query.filter_by(...).first()`
4. Access order via `item.order` relationship instead of separate query

**Result**: Function now uses direct database access (SQLAlchemy ORM) which works correctly

## WHY THIS FIXES THE ISSUE ✅

**Supabase REST API Issue**:
- REST API requires proper authentication and RLS policies
- Current API key may not have full read access
- RLS policies may be blocking anonymous access
- This causes ALL product queries to return empty

**SQLAlchemy ORM Solution**:
- Connects directly to PostgreSQL database
- Uses database credentials (not REST API)
- Bypasses RLS policies
- Always returns correct data

## ADDITIONAL NOTES ✅

### Supabase REST API vs SQLAlchemy ORM

**When to use Supabase REST API**:
- Mobile app API endpoints
- External integrations
- When you need REST API features

**When to use SQLAlchemy ORM**:
- Internal Flask functions
- Complex queries with relationships
- When REST API has permission issues
- For better performance

### Current Implementation:
- **add_to_cart_ajax**: Uses Supabase REST API (for mobile compatibility)
- **get_available_stock**: Uses SQLAlchemy ORM (for reliability)
- **Most routes**: Use SQLAlchemy ORM (existing code)

This hybrid approach gives us:
✅ Reliable stock calculations
✅ Mobile API compatibility
✅ Best of both worlds
